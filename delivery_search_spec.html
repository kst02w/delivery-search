Delivery Search 詳細設計書

対象ファイル: delivery_search_v4 (index.html) ／ 作成: ECCO Japan IT ／ バージョン: v4

1. 概要

ECCOジャパン国内80店舗向けの出荷明細照会ツール。SAPから出力した出荷明細ExcelをGitHub Pagesにホストし、店舗スタッフがブラウザから自店舗の出荷内容（DN番号・商品・サイズ別数量）を確認できる。

項目内容
公開URLhttps://kst02w.github.io/delivery-search/
ホスティングGitHub Pages（静的サイト・無料）
利用者国内約80店舗のスタッフ（閲覧のみ）
更新者本部IT担当者（週1回）
使用ライブラリSheetJS (xlsx 0.18.5) — Excelパース用
認証なし（URL共有によるアクセス）
画像サーバー社内DNS限定 — 社外NW/VPNなしでは画像不可

2. ファイル構成

delivery-search/                 # GitHubリポジトリルート
├── index.html                   # ツール本体（全ロジック含む単一ファイル）
├── index.json                   # 週リスト定義ファイル（本部が週次更新）
├── current.xlsx                 # 今週の出荷明細（本部が毎週上書き）
└── archive/                     # 過去週のアーカイブフォルダ

    ├── 2026_W12.xlsx
    └── 2026_W11.xlsx
index.htmlとデータファイルは同一リポジトリルートに配置。相対パスは ./ で参照。

index.html は単一ファイルで4層構造を持つ。

UI層（タブ・フォーム）

担当要素

要素ID種類役割

データ取得層

3つのデータ入力経路を持つ。いずれも最終的に allRows（グローバル配列）に格納し initControls() を呼ぶ。

処理層

render() 内で Store フィルタ → DN集計 → サイズ集計 → HTML文字列生成の順に処理する。

処理内容
StoreフィルタallRows.filter(r => r['Store'] === store)

表示層

5. 関数仕様

initWeekList()

項目内容
種別async function
呼び出し元ページロード時（スクリプト末尾で即時実行）
引数なし
戻り値なし

処理フロー

week-badge を「読み込み中」に → fetch(DATA_BASE + 'index.json') → week-sel にオプション生成 → 先頭週を自動選択 → loadWeekFile() 呼び出し

エラー処理

fetch失敗またはHTTPステータス非200の場合、week-badge を「index.jsonが見つかりません」（赤バッジ）に変更し処理終了。

try {
  const res = await fetch(DATA_BASE + 'index.json', { cache: 'no-cache' });
  if (!res.ok) throw new Error('HTTP ' + res.status);
  const json = await res.json();
  // ...
} catch (e) {
  badge.className = 'badge-err';
  badge.textContent = 'index.json が見つかりません';
}

loadWeekFile()

項目内容
種別async function
呼び出し元week-sel の onchange / initWeekList()
引数なし（week-sel.value から取得）
戻り値なし

処理フロー

week-sel.value を取得 → fetch(DATA_BASE + file) → res.arrayBuffer() → parseXlsx(buf) 呼び出し

エラー処理

fetch失敗時は week-badge を「ファイル取得失敗: {エラーメッセージ}」に変更。

parseXlsx(buf)

項目内容
種別function
呼び出し元loadWeekFile() / loadFile()
引数buf: ArrayBuffer — Excelファイルのバイナリ
戻り値なし（allRows を更新し initControls() を呼ぶ）

処理内容

const wb = XLSX.read(buf, { type: 'array' });
const ws = wb.Sheets[wb.SheetNames[0]];   // 先頭シートのみ対象
const json = XLSX.utils.sheet_to_json(ws, { defval: '' });
allRows = json.map(r => {
  const obj = {};
  Object.keys(r).forEach(k => obj[k.trim()] = String(r[k]).trim());
  return obj;
});
先頭シートのみ処理。ヘッダー行は sheet_to_json が自動認識（1行目をキーとして使用）。全値を文字列に変換（trim済み）。

parseTSV(text)

項目内容
種別function
呼び出し元loadPaste()
引数text: string — TSV形式のテキスト（1行目がヘッダー）
戻り値Array<Object> — ヘッダーをキーとするオブジェクト配列

処理内容

const lines = text.trim().split('\n');
if (lines.length < 2) return [];         // ヘッダーのみの場合は空配列
const headers = lines[0].split('\t').map(h => h.trim());
return lines.slice(1).filter(l => l.trim()).map(l => {
  const cols = l.split('\t');
  const obj = {};
  headers.forEach((h, i) => obj[h] = (cols[i] || '').trim());
  return obj;
});

initControls()

項目内容
種別function
呼び出し元parseXlsx() / loadPaste() — allRows 更新後に必ず呼ぶ
引数なし（allRows グローバル変数を参照）
戻り値なし

処理内容

allRows から Store列のユニーク値を抽出し昇順ソートして store-sel のオプションを再生成。controls-card を表示し result-card を非表示にリセット。

const stores = [...new Set(allRows.map(r => r['Store']).filter(Boolean))].sort();
// store-sel にオプション生成
// controls-card.style.display = ''
// result-card.style.display = 'none'

render()

項目内容
種別function
呼び出し元store-sel の onchange
引数なし（store-sel.value と allRows グローバル変数を参照）
戻り値なし（tbl-area の innerHTML を更新）

集計ロジック詳細

ステップ処理使用列
1. Storeフィルタ選択Storeと一致する行のみ抽出Store
2. サイズ列特定GrV列の値が /^\d{2}$/ にマッチするユニーク値を収集・数値昇順ソートGrV
3. byDN構築DN no をキーに {date, mats{}} を生成DN no, ReqDelDate
4. マテリアル集計Material または Material Number をキーに sizes{} を構築Material, Material Number
5. サイズ数量加算GrV × Quantity/Qty を sizes に加算GrV, Quantity, Qty
6. HTML生成DN行 → マテリアル行の順でHTML文字列を構築しtbl-areaに挿入—

Quantity列フォールバック

const qty = parseInt(r['Quantity'] || r['Qty'] || 1) || 1;
Quantity列が存在しない場合はQty列を参照。どちらもない場合は1として扱う。

getImageUrl(matCode)

項目内容
種別function（VBAの InsertPictureInRange のURL生成ロジックをJS移植）
呼び出し元render() 内のマテリアル行生成時
引数matCode: string — Material列の値
戻り値string | null — 画像URL、空文字の場合はnull

ロジック

function getImageUrl(matCode) {
  const code = (matCode || '').trim();
  if (!code) return null;
  // 10桁の場合は先頭に0を付加（VBAの Len = 10 判定と同じ）
  if (code.length === 10) {
    return 'http://image/product/0' + code + '/*/thumbnail?nodefault=true';
  }
  return 'http://image/product/' + code + '/*/thumbnail?nodefault=true';
}
画像サーバーは社内DNS限定。社外NW（ポケットWi-Fi等）では名前解決できず画像非表示。imgタグのonerrorでNo imgプレースホルダーに自動切り替え。

switchTab(t)

項目内容
引数t: 'auto' | 'paste' | 'file'
処理tab-{t} に active クラスを付与、pane-{t} を表示。他を非表示。

openLightbox(src) / closeLightbox()

関数処理
openLightbox(src)lightbox-img の src を設定し lightbox に open クラスを付与（display:flex）
closeLightbox()lightbox から open クラスを除去（display:none）

lightbox 要素自体に onclick="closeLightbox()" が設定されているため、背景クリックで閉じる。

6. データフロー

自動読み込みフロー

ページ表示 → initWeekList() → fetch index.json → week-sel 生成 → loadWeekFile() → fetch current.xlsx → parseXlsx() → allRows 更新 → initControls() → Store選択 → render() → テーブル表示

Excelアップロードフロー

ファイル選択 → loadFile(input) → FileReader.readAsArrayBuffer → parseXlsx(buf) → allRows 更新 → initControls() → Store選択 → render() → テーブル表示

コピペ入力フロー

TSV貼り付け → loadPaste() → parseTSV(text) → allRows 更新 → initControls() → Store選択 → render() → テーブル表示

7. データ仕様

index.json フォーマット

{
  "weeks": [
    { "label": "2026 W14（最新）", "file": "current.xlsx" },
    { "label": "2026 W13",         "file": "archive/2026_W13.xlsx" },
    { "label": "2026 W12",         "file": "archive/2026_W12.xlsx" }
  ]
}
フィールド型説明
weeksArray週エントリの配列。先頭が最新週として自動選択される。
weeks[].labelstringドロップダウンに表示される週の名称。
weeks[].filestringDATA_BASEからの相対パス。current.xlsx または archive/YYYY_WXX.xlsx。
週次更新時は先頭に新エントリを追加し、旧「（最新）」表記を外す。

Excel（出荷明細）フォーマット

SAPから出力されたExcelの先頭シートを処理する。以下の列名を参照。

列名必須説明
Store必須店舗コード。Store選択ドロップダウンの生成に使用。
DN no必須デリバリーノート番号。DN行のグルーピングキー。
ReqDelDate推奨希望納品日。DN行に表示。
Material推奨マテリアルコード。画像URL生成にも使用。
Material Number推奨マテリアル番号（商品名等）。テーブル表示用。
GrV必須サイズコード（2桁数字）。サイズ列の生成に使用。
Quantity または Qty推奨数量。どちらかが存在すれば良い。なければ1として扱う。

allRows グローバル変数の構造

全データ入力経路で共通のグローバル配列。型は Array<Object>。

allRows = [
  {
    "Store":           "NSG",
    "DN no":           "826852918",
    "ReqDelDate":      "46106",
    "Material":        "20682301001",
    "Material Number": "ECCO Cozmo W Black",
    "GrV":             "37",
    "Quantity":        "2"
  },
  // ... 以降同構造のオブジェクトが続く
]
全値は文字列型。数値比較が必要な場合は parseInt() や Number() で変換する。

8. 制約・注意事項

項目制約内容対応策
画像サーバー社内DNS限定URL。社外NWから名前解決不可。onerrorでNo imgに自動切り替え。VPN接続で解決可。
SharePoint CSPSharePointのiframe sandboxによりJS実行不可。GitHub PagesまたはAzure Static Web Appsでホスティング。
GitHub Pages公開リポジトリがPublicのため出荷データが外部公開。Azureへの移行後はEntra ID認証で社内限定化予定。
Excel形式先頭シートのみ処理。ピボットテーブル形式は非対応。SAP出力の生データ形式（フラットリスト）を使用すること。
ファイルサイズGitHub Pages: 1ファイル100MB上限、リポジトリ1GB上限。通常の出荷明細では問題なし（数百KB程度）。
ブラウザ互換IE非対応（async/await, fetch, Spread構文使用）。Chrome/Edge/Safari最新版で動作確認済み。
